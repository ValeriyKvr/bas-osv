# ОСВ з 1С → PostgreSQL → дашборд

Автоматична викачка **оборотно-сальдової відомості (ОСВ)** з 1С:Підприємство
напряму через COM, завантаження в PostgreSQL і перегляд у вигляді ієрархічної
ОСВ (як ручне вивантаження з 1С) у Streamlit-дашборді.

## Структура проєкту

```
.
├── config.py            # усі налаштування (база 1С, організація, PostgreSQL, періоди)
├── main.py              # CLI: 1С → Excel/PostgreSQL
├── dashboard.py         # точка входу Streamlit-дашборда (навігація)
├── views/               # сторінки дашборда
│   ├── compare.py       #   порівняння двох періодів (KPI, рух грошей, контрагенти)
│   ├── osv.py           #   ОСВ одного періоду з розгортанням ієрархії
│   └── load.py          #   викачка обраних періодів з 1С прямо з дашборда
├── osv1c/               # ядро пайплайну (пакет)
│   ├── connector.py     #   COM-підключення до 1С, запити ОСВ та деталізації
│   ├── periods.py       #   період = квартал або місяць (тег, межі, назва)
│   ├── db.py            #   PostgreSQL: схема + завантаження
│   ├── report.py        #   побудова ієрархічної ОСВ із БД (згортання по дереву)
│   ├── compare.py       #   порівняння періодів: ОСВ A/B, статті ДДС, контрагенти
│   └── excel.py         #   запис ОСВ у Excel
├── data/samples/        # зразки вивантажень (1q.xlsx — еталон формату)
├── exports/             # згенеровані Excel-файли (osv_<рік>-Q<кв>.xlsx)
└── backup/              # бекап бази 1С (1Cv8.dt)
```

## Передумови

- 1С:Підприємство 8.3 (файлова база), COM-конектор зареєстровано один раз:
  ```powershell
  regsvr32 "C:\Program Files\1cv8\<версія>\bin\comcntr.dll"
  ```
  (потребує прав адміністратора; розрядність dll = розрядності Python)
- PostgreSQL (локально), база створюється автоматично при першому `--db`.
- Залежності:
  ```powershell
  pip install -r requirements.txt
  ```

## Налаштування

Усе в [`config.py`](config.py):
- `C1_BASE_PATH`, `C1_USER`, `C1_PASSWORD` — файлова база 1С;
- `C1_ORGANIZATION` — організація для ОСВ (у базі їх може бути кілька);
- `PG_*` — підключення до PostgreSQL;
- `PERIODS` — періоди за замовчуванням (можна перекрити прапором `--period`).

## Використання

**Викачати ОСВ у БД (і Excel):**
```powershell
python main.py --db --period 2025-Q1            # квартал
python main.py --db --period 2026-M01           # місяць (січень 2026)
python main.py --db --period 2025-Q1 2026-Q1    # кілька періодів
python main.py --db                              # періоди з config.PERIODS
python main.py --no-db --period 2025-Q1          # тільки Excel
```

Період — квартал (`2026-Q1`) або місяць (`2026-M03`). Можна викачувати
й з дашборда: сторінка **⚙️ Завантаження з 1С** — обираєте роки та
квартали/місяці, кнопка робить те саме, що `main.py --db`.

**Відкрити дашборд:**
```powershell
streamlit run dashboard.py
```

Дашборд має дві сторінки:

- **📈 Порівняння періодів** (відкривається першою) — два квартали поруч,
  для презентації фінансовому директору:
  - KPI: дохід (Кт 70), собівартість (Дт 90), адмін+збут (Дт 92+93),
    фінрезультат періоду (зміна сальдо 44), надходження/платежі грошей,
    залишок грошей, борг покупців — кожен з Δ та Δ% до бази;
  - **Рух грошей за статтями** (субконто банківських виписок, рах. 30/31/33):
    на що платимо і звідки надходять гроші, з графіком топ-статей;
    внутрішні перекази між власними рахунками виключаються прапорцем;
  - **Обороти по рахунках**: ієрархічна ОСВ з колонками обох періодів,
    Δ і Δ% (показник на вибір: оборот Дт / Кт / сальдо кін.);
  - **Постачальники (631)** і **Покупці (361)**: обороти та борги по
    контрагентах у двох періодах, топ-10 на графіку;
  - кнопка **експорту всього порівняння в один Excel** (5 аркушів:
    KPI, рух грошей, обороти, постачальники, покупці).
- **📊 ОСВ (відомість)** — класична ОСВ одного періоду з розгортанням
  рівнів і деталізацією по субконто.
- **⚙️ Завантаження з 1С** — викачка обраних періодів (кварталів або
  місяців) у PostgreSQL без консолі; параметри підключення до 1С можна
  змінити прямо на сторінці (типово — з config.py). Працює лише на
  машині, де встановлена 1С і доступна база.

Для порівняння потрібні щонайменше два завантажені періоди, напр.:
```powershell
python main.py --db --period 2025-Q1 2026-Q1
```

## Як це працює

1. `connector.py` підключається до 1С через `V83.COMConnector` і запитом до
   `РегистрБухгалтерии.Хозрасчетный.ОстаткиИОбороты` дістає ОСВ за період
   (з фільтром по організації) та деталізацію в розрізі субконто.
2. `db.py` вантажить це в PostgreSQL: `accounts` (план рахунків з ієрархією),
   `osv_summary` (залишки по рахунках), `osv_detail` (субконто).
3. `report.py` згортає залишки по дереву рахунків у класичну ОСВ (з підсумками
   по класах і збалансованим рядком «Разом»).
4. `dashboard.py` показує її з розгортанням рівнів, пошуком і деталізацією.

> **Дати в запитах 1С передаються літералом `ДАТАВРЕМЯ`**, а не Python-datetime —
> інакше pywin32 зсуває межі періоду на часовий пояс (реформація 31.12 «затікає»
> в Q1, закриття 31.03 «випадає»).
